#### 策略模式
1. 策略模式定义了  算法族  ,分别封装起来,让他们之间可以     相互替换。（定义了一组同类型的算法，在不同的类中封装起来，每种算法可以根据当前场景互换，
从而使得算法的变化 独立于使用它们的客户端（算法的调用者）
##### 使用场景
当 if-else 代码量比较大的时候，后续的扩展和维护会变得复杂且容易出错。阿里《Java开发手册中》，超过3层的if-else的逻辑判断代码可以使用卫语句（多个if），策略模式、状态模式来实现
